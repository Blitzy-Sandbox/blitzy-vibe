EXECUTIVE SUMMARY

Three capabilities, one cohesive startup pipeline

A numbered prompt is presented before any LLM backend is instantiated, the Anthropic backend resolves its API key through a deterministic env → config → prompt chain, and a SessionManager writes per-repo, per-branch JSON after every turn with auto-compaction at 80% of the active provider’s configured context limit.

ARCHITECTURE · §0.5.4

End-to-end startup flow

Argparse → resume check → provider picker → key resolution → backend factory

flowchart TD
    A[python -m vibe] --> B[argparse]
    B --> C{--resume present?}
    C -- Yes --> D[git_context.detect repo, branch]
    D --> E[SessionManager.list_sessions]
    E --> F{sessions found?}
    F -- Yes --> G[session_picker.select_session]
    G --> H[Load messages + provider from session JSON]
    H --> Z[Instantiate backend via factory]
    F -- No --> I[Print no-sessions notice]
    I --> J{--provider present?}
    C -- No --> J
    J -- Yes --> K[provider_string_to_backend]
    J -- No --> L[provider_picker.select_provider]
    L --> K
    K --> M[api_key_prompt.resolve_or_prompt]
    M --> N{Key resolved?}
    N -- No --> X[raise MissingAPIKeyError, exit]
    N -- Yes --> Z
    Z --> Y[run_cli loop]

Capability A · 01 of 03

Interactive provider picker before any backend constructs

Provider Selection · Capability A

Numbered prompt — verbatim per AAP §0.2.2

Select LLM provider: [1] Blitzy (default) [2] Mistral [3] Anthropic >
  • Empty input (Enter) selects [1] Blitzy — the default
  • Case-insensitive token matching: 1 | blitzy | 2 | mistral | 3 | anthropic
  • --provider {blitzy,mistral,anthropic} skips the prompt entirely
  • --resume opens the session picker; falls through to the provider picker when no sessions exist for the current repo and branch

Rule 4 — Provider selection MUST be displayed before any backend is instantiated; verified by asserting no backend constructor is called if selection is cancelled.

Capability B · 02 of 03

Direct Anthropic SDK with deterministic key resolution

Anthropic Backend · Capability B

Deterministic three-tier key resolution

flowchart LR
    Start[AnthropicBackend.__init__] --> Env{ANTHROPIC_API_KEY env set?}
    Env -- Yes --> Use[Use env value]
    Env -- No --> Cfg{config.anthropic_api_key set?}
    Cfg -- Yes --> Use
    Cfg -- No --> Prompt[Interactive getpass prompt]
    Prompt --> Empty{Empty input?}
    Empty -- Yes --> Err[raise MissingAPIKeyError, exit]
    Empty -- No --> Save{Save to ~/.blitzy/config.toml?}
    Save -- Yes --> Persist[tomli_w write]
    Save -- No --> Use
    Persist --> Use
    Use --> Client[Construct anthropic.AsyncAnthropic]
    Client --> Done[Backend ready]
  • Default model: claude-sonnet-4-6 (pinned snapshot, AAP §0.10.2)
  • Config field anthropic_model overrides the default
  • Streaming via client.messages.stream() — unchanged from the existing implementation
  • Existing pin anthropic>=0.100.0 retained (boundary directive: additive extension only)

Rule 10 — MissingAPIKeyError MUST be raised and the agent MUST exit if the user declines interactive key entry; verified by test mocking declined input for each provider.

Session Persistence · Storage

Filesystem layout — one JSON document per session

~/.blitzy/ ├── config.toml ├── .env └── sessions/ ├── blitzy-agent/ │ ├── main/ │ │ ├── a1b2c3d4-....json │ │ └── f0e1d2c3-....json │ └── feature-branch/ │ └── 99887766-....json └── _unknown/ └── _unknown/ └── 11223344-....json

Record schema

{
  "session_id": "<uuid4>",
  "created_at": "<ISO 8601>",
  "provider": "<blitzy|mistral|anthropic>",
  "repo": "<str>",
  "branch": "<str>",
  "messages": [...],
  "compacted_summary": "<str|null>"
}

Rule 6 — Session files MUST be written (full overwrite) after every turn at ~/.blitzy/sessions/{repo_name}/{branch_name}/.

Session Persistence · Compaction · §0.5.5

Per-turn loop with 80%-threshold auto-compaction

flowchart LR
    A[User input] --> B[Append LLMMessage to session.messages]
    B --> C[AutoCompactMiddleware.before_turn]
    C --> D{tokens >= 80 percent of context_limit?}
    D -- Yes --> E[SessionManager.compact session, limit]
    E --> F[backend.complete_streaming]
    D -- No --> F
    F --> G[Append assistant LLMMessage]
    G --> H[SessionManager.save full overwrite]
    H --> I[observability.metrics.increment turns_completed]
    I --> J[Wait for next input]
Token estimator: len(json.dumps(messages)) // 4 Defaults (overridable via [context_limits]): blitzy=128000, mistral=32000, anthropic=200000

Rule 7 — Auto-compaction MUST trigger when the token estimate exceeds 80% of the active provider’s context limit; MUST replace compacted messages with a single system summary; MUST preserve the most recent messages verbatim.
